# UI Polish Plan — All Pages

## Chat Message Layout (ChatMessage.tsx + globals.css)

### Current problem
- Both user and assistant messages are left-aligned with basic bubble styles
- `message-user` has `margin-left: 2rem` but stays left-aligned
- `message-assistant` has `margin-right: 2rem` but stays left-aligned
- `max-w-3xl` on both restricts width unnecessarily
- No visual distinction between roles

### Fix
Rewrite `ChatMessage.tsx`:
- **User messages**: Right-aligned using `flex justify-end` on wrapper. Cyan-tinted background. Avatar "U" on right side. Border radius `12px 12px 4px 12px`.
- **Agent messages**: Left-aligned. Dark subtle background with cyan left border. Agent avatar/name on left. Border radius `12px 12px 12px 4px`.
- Both: `max-w-[75%]` for comfortable reading width, proper `p-4` padding

Update `globals.css`:
- Remove old `.message-user` / `.message-assistant` CSS rules (they use margins incorrectly)
- Replace with new flex-based approach (handled in component JSX)

## Chat Message Area (ChatInterface.tsx)

### Current problem
- `p-6` padding on message container — too much on small screens
- Messages list has `space-y-4` but individual messages also have padding creating double gaps

### Fix
- Reduce to `p-4` on container
- Messages control their own spacing via margin
- Proper bottom padding to account for input area

## Chat Input Area (ChatInput.tsx)

### Current problem
- Provider selector buttons look cramped with tiny `text-[11px]`
- Textarea has `pr-24` which can overlap with send button on narrow views
- `max-w-3xl mx-auto` centers input unnecessarily — should span full width

### Fix
- Provider selector: cleaner row with proper spacing, slightly larger text
- Remove `max-w-3xl mx-auto`, use `px-6` padding instead
- Send button positioning improved

## Agent Header Bar (ChatInterface.tsx)

- Add subtle gradient or solid background
